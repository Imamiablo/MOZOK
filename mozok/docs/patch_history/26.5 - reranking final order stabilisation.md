# 26.5 - Reranking Final Order Stabilisation

## Summary

Stabilises the final prompt ordering for reranked memories.

## Changes

- Adds a small helper that reads transient `metadata["_reranking"]["final_score"]` from `MemorySearchResult` objects.
- Orders surviving semantic, episodic, and raw memories by reranking final score after safe deduplication.
- Keeps unscored memories stable and places them after scored memories.
- Makes final prompt memory order, debug `sections`, `used_memory_ids`, and token-budget tail trimming follow the same order.
- Removes old patch-marker comments around the reranking debug method.

## Notes

This does not modify SQL, FAISS, stored memory metadata, or memory access counters. Reranking metadata remains request-local.
