# Patch 14 - FastAPI TestClient smoke/contract tests

This patch adds a tiny integration-test layer for the FastAPI app.

## Added

- `tests/integration/test_fastapi_openapi_contract.py`
- `httpx==0.27.2` in `requirements-dev.txt`

## What the tests check

- FastAPI app imports correctly.
- `/openapi.json` is generated successfully.
- Core routes are registered:
  - `POST /chat`
  - `POST /memories`
  - `POST /debug/context`
- `/debug/context` request schema still exposes the context/debug/budget controls.

## What these tests deliberately do not check yet

- PostgreSQL integration
- FAISS integration
- Ollama/LLM calls
- Real `/chat` response generation
- Real `/debug/context` retrieval with database records

Those belong to a later integration-test layer with a test database and mocked LLM.
