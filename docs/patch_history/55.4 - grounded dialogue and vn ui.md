# 55.4 - Grounded Dialogue and VN UI

## Summary

This patch tightens the sandbox contract between LLM prose and the game-owned physical world:

- added `scene_validation.py` with scene grounding, inventory/object visibility context, and dialogue stage-direction validation
- chat prompts now include speaker inventory, nearby objects, legal interactions, and the rule that LLM prose may not assert unvalidated physical state
- agent chat replies and tick-selected dialogue are grounded before they are shown or logged
- unowned item gestures such as `*fidgets with the camp knife*` are rewritten generically instead of patched by character/item name
- added tests for unowned item rewrites, owned item allowance, and scene grounding
- bottom HUD now has tabs for Conversation, Inventory, Agent, and Memory
- free-text chat now uses a lower visual-novel-style dialogue panel with portrait, latest line, input, and history scrolling
- agent-initiated nearby conversations can open the chat panel automatically so the player feels addressed, not merely passed by

## Why

The LLM can write evocative dialogue, but physical state belongs to the game engine. A character can glance toward a visible object, but holding, using, opening, handing, eating, tying, or fidgeting with a prop must be supported by inventory, proximity, or a recent validated action.

This is a generic guardrail for any future scenario, item, and character. It is not an Alice/knife exception.
