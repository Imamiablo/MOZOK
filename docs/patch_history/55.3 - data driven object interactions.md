# 55.3 - Data Driven Object Interactions

## Summary

This patch moves normal world-object use onto the same generic foundation as item capabilities:

- added `object_type`, `sprite`, `properties`, `render`, and `interaction_defs` to `WorldObject`
- scenario objects can now define `interactions` as data effects instead of only string names
- added `object_effects.py`, a generic interpreter for object interactions and inventory item use
- converted campfire, food crate, shelter, spring, radio, cave, berries, pickup items, journal page, and lockbox to data-defined interactions
- `interact_with_object()` and agent `_agent_use_object()` now call the generic interaction interpreter
- agent affordances can score new interactions from `affordance_tags` and `actor_need_delta`
- semantic parser prompts now build known object/id/kind/interactions context from the current world instead of a hardcoded island object list
- speech parsing can carry `target_object_id` directly
- storylet protection checks can use object tags, so a future shelter/fire/bed object can satisfy a condition without a Python branch

## Why

The previous pass made item capabilities data-driven, but basic interactions still knew too much about the island. This pass makes the lower-level object model suitable for editor-created objects and future drag-and-drop PNG workflows:

`sprite + tags + interactions + effects -> visible object with simulation meaning`
