# 54.2 - Island Prototype MOZOK API Bridge Hotfix

Improves the optional live MOZOK API bridge for the island prototype.

## Fixed

- Runtime tick payload now sends both `perception_events` and valid `sensory_inputs`.
- Action planning is nudged toward `game_command` actions instead of always selecting the baseline `speak` action.
- `move_to_object` actions now get a game-side `object_id` target when MOZOK returns a generic adapter intent.
- API fallback reasons are now visible in the in-game event log instead of being swallowed silently.
- Start-up logs whether the game is using offline brain or MOZOK API mode.

## Usage

```powershell
$env:MOZOK_GAME_USE_API="1"
$env:MOZOK_API_BASE_URL="http://127.0.0.1:8000"
.\.venv\Scripts\python.exe -m mozok_game.main
```

If your API is on another port, change `MOZOK_API_BASE_URL`.
