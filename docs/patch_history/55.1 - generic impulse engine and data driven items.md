# 55.1 - Generic Impulse Engine And Data Driven Items

## Summary

This patch moves the island sandbox closer to a reusable simulation foundation:

- item definitions now live in `mozok_game/data/items/items.json`
- world objects can declare `capability_accepts` and `capability_effects` in scenario data
- capability execution can apply data-driven effects such as state updates, item grants, item consumption, claims, need deltas, social deltas, flashes, and event logs
- world events now carry structured fields for actor, target, item, location, witnesses, visibility, and reliability
- storylets are loaded from `mozok_game/data/storylets/storylets.json`
- a first `ImpulseEngine` maps pressure axes, traits, social state, and recent event tags into candidate impulses
- drama atoms live in `mozok_game/data/drama_atoms/core_atoms.json`

## Why

The previous foundation pass removed direct Alice/Boris/Mira checks and added traits, commitments, item capabilities, and pressure. This pass makes the next step: scenario authors can add more behaviour through data, while the engine remains responsible for validation and consequences.

## Verification

- `mozok_game syntax check passed`
- `mozok_game smoke tests passed`

## Notes

This is still not a full LLM scene-weaver. It is the deterministic substrate it will need:

`world event -> pressure field -> agent appraisal -> impulse candidates -> validated action`

The next natural step is to let MOZOK/LLM choose between generated impulse candidates and compose richer dialogue while the game engine keeps ownership of physical state.
