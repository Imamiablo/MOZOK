# 46.1 — Reflection Learning V2 proposal hotfix

## Summary

Fixes the Reflection Learning V2 path so reflected goal, entity-state, and belief-revision hints create first-class safe change proposals instead of being folded only into the older generic `reflection` proposal family.

## Fixed

- `ReflectionService.reflect()` now emits:
  - `reflection_goal_update`
  - `reflection_entity_state_update`
  - `reflection_belief_revision`
- Goal update hints now produce reviewed `update_goal` operations.
- Entity-state update hints now produce reviewed `update_entity_state` operations.
- Belief revision triggers now produce reviewed metadata-backed follow-up proposals.
- Auto-apply policy is no longer restricted to only the generic `reflection` proposal type.
- Removed stray duplicated hint fields from `ReflectionBeliefRevisionTrigger` schema.

## Tests

Updated `tests/unit/test_v44_48_runtime_learning.py` to assert both proposal families and the underlying operation types.

## Notes

This is a hotfix only. It does not change the wider V44–48 execution, world-event, self-model, or belief-graph APIs.
