# 26.2 - Final Reranking Pipeline Debug Hotfix

## Purpose

Patch 26 added deterministic memory reranking and Patch 26.1 exposed the reranking report on `ContextPackage`. However, the final `pipeline_steps[-1]` entry did not include the same reranking report. This made the debug output inconsistent and broke tests that expected the final prompt step to explain which reranked memories remained in the final prompt.

## Changes

- Adds `memory_reranking` to the `final_prompt` pipeline step in `ContextPackage.pipeline_steps()`.
- Keeps the top-level debug field `memory_reranking` unchanged.
- Does not modify SQL, FAISS, retrieval, scoring, or prompt assembly behaviour.
