# 26.5 - Live Search Reranking Wrapper

## Summary

Adds a robust wrapper around `MemoryService.search` so live API/debug responses include reranking explanations even when the older search body returns plain `MemorySearchResult` objects without `_reranking` metadata.

## Why

Patch 26 added the reranker and debug display. Swagger testing showed that `/debug/context` still returned an empty `memory_reranking` list in the live path. That means the debug layer was ready, but live search results were not consistently carrying reranking metadata.

## Behaviour

- Does not mutate SQL.
- Does not mutate FAISS.
- Reranks only the outgoing search results.
- Adds `_reranking` only to response metadata.
- Preserves existing MemoryService behaviour, including access updates from the original search method.
- Uses relation signals when available.
