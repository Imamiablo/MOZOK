# Patch 8.2 - Read-only debug context

`POST /debug/context` now builds the same context as chat, but does not update memory access statistics.

## Changed

- `mozok/memory/service.py`
  - `search(..., update_access=True)` supports disabling access_count updates.

- `mozok/context/context_builder.py`
  - `ContextBuilder.build(..., update_memory_access=True)` passes this option into semantic/episodic/raw searches.

- `mozok/api/main.py`
  - `/debug/context` calls `ContextBuilder.build(..., update_memory_access=False)`.

## Result

- `/chat` still updates `access_count` because default is `True`.
- `/debug/context` no longer increments `access_count` while inspecting context.
