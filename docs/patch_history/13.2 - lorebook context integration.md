# Patch 16 - Lorebook context integration

This patch finishes the Lorebook MVP by connecting existing Lorebook storage/API to the normal LLM context pipeline.

## What changed

- `ContextBuilder.build(...)` can now fetch lorebook entries for an agent/world.
- `ContextPackage.to_system_prompt()` injects a `Lorebook / world knowledge` section before memories.
- `/debug/context` accepts lorebook options and shows `sections.lorebook` plus `used_lorebook_entry_ids`.
- `/chat` accepts the same lorebook options and passes them into `BotCore`.
- `ChatResponse` reports which lorebook entries were used.
- Token-budget trimming can remove lorebook items after memory buckets but before short-term/core context.
- Added unit tests for prompt injection, restricted lore access, and lorebook token-budget trimming.

## Important defaults

- `world_id = "default"`
- `lorebook_limit = 10` in `/chat` and `/debug/context`
- `include_public_lore = true`
- `include_narrator_only_lore = false`

For normal NPCs/assistants, keep `include_narrator_only_lore = false`.
For narrator/game-master agents, set it to `true` when needed.

## How to verify manually

1. Run database init if needed:

```powershell
.\.venv\Scripts\python.exe scripts\init_db.py
```

2. Run tests:

```powershell
.\.venv\Scripts\python.exe -m pytest
```

3. Use Swagger:

```text
http://127.0.0.1:8000/docs
```

4. Create lore with `POST /lorebook/upsert`.
5. Optionally link restricted lore with `POST /agents/{agent_id}/lorebook/knowledge`.
6. Check `POST /debug/context` and confirm:

```text
sections.lorebook
used_lorebook_entry_ids
full_prompt contains the lorebook section
```
