# 26.4 - Live Reranking Metadata Wiring

## Purpose

Patch 26 added the deterministic reranker and debug report plumbing, but live
`/debug/context` responses could still show an empty `memory_reranking` list.
The reason was that real `MemoryService.search()` results were still using the
old lightweight ranking block and did not attach `_reranking` metadata to
`MemorySearchResult.metadata`.

## Changes

- Replaced the old lightweight in-method ranking in `MemoryService.search()` with
  `MemoryReranker`.
- Added `_memory_relation_signals_for_reranking()` so direct memory relations can
  boost memories connected to goals, lorebook entries, entity states, entities,
  factions, quests, locations, or objects.
- Attached reranking explanations only to outgoing search-result metadata.
- Confirmed that SQL memory metadata is not mutated with `_reranking` debug data.
- Ensured the final debug pipeline step exposes `memory_reranking` when needed.

## Behaviour

The reranker remains deterministic and transparent. It does not call an LLM and
it does not modify SQL, FAISS, access counters, or memory contents.

## Notes

This patch keeps the old vector score in the public `score` field for backwards
compatibility. The final reranking score and score parts are available in:

```json
metadata._reranking
```
