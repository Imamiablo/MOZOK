# 33.2 - Cognitive Field prompt fallback hotfix

## Summary

This hotfix ensures the Cognitive Field prompt section is rendered whenever a Cognitive Field report exists, even if the selected broadcast does not contain a populated `working_memory_line`.

Previously, `/debug/context` could expose a valid `cognitive_field` debug report while `ContextPackage.to_system_prompt()` skipped the broadcast section because it checked only `broadcast.working_memory_line`. This made the debug state and prompt state disagree.

## Changed files

- `mozok/context/context_builder.py`

## Behaviour

- If `enable_cognitive_field=true`, the final prompt now includes the `Cognitive Field / broadcast focus for this turn` section whenever `context.cognitive_field` is present.
- If a winner exists, the section uses `working_memory_line` as before.
- If no winner passes the threshold, the section still renders a safe fallback summary/policy line, keeping prompt/debug behaviour consistent.
- The layer remains read-only and does not persist memories, goals, skills, entity states, relations, or FAISS changes.

## Tests

This directly fixes `tests/unit/test_cognitive_field_mvp.py::test_context_builder_can_inject_cognitive_broadcast_into_debug_and_prompt`.
