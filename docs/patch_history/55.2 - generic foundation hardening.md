# 55.2 - Generic Foundation Hardening

## Summary

This patch closes more of the gap between the island demo and a generic sandbox foundation:

- removed `TARGET_PRIMITIVES` and island-specific capability execution branches for cave, lockbox, and berries
- made item definitions truly data-owned by keeping the Python fallback empty
- generalized capability affordance generation so agents scan item capabilities against object-declared accepts/tags instead of looking for one named cave/box/berry path
- added `truth_status` and `idempotency_key` to world events
- sends both `world_event_id` and per-agent `perception_id` to MOZOK so global events and agent perceptions are distinct
- added commitment sync helpers so legacy follow/target command fields are derived cache for UI/backward compatibility
- added tests for invalid capability rejection, event idempotency payloads, pressure bounds/decay, and commitment expiry/interruption cache cleanup

## Why

The goal is to let scenario data define objects, items, and consequences while the Python engine stays a small validator/executor. New objects should not require a new `if obj.kind == ...` branch just to accept a basic primitive.
