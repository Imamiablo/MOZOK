# 27.1 - Project review cleanup and brain pack memory import hotfix

## Summary

Version 27.1 is a review/cleanup hotfix. It keeps version 27's procedural skill selector forwarding work, but fixes project packaging and the brain-pack memory import path discovered during review.

## Changed

- Added root `.gitignore` to keep generated caches, virtual environments, logs, FAISS runtime files, installer EXEs, and large local model artefacts out of commits.
- Added `requirements.txt`, `requirements-dev.txt`, `.env.example`, `README.md`, and `ROADMAP.md` so the project can be recreated and tested from a fresh checkout.
- Consolidated stray patch-history files from `mozok/docs/patch_history` into the canonical `docs/patch_history` folder.
- Made `sentence-transformers`, `openai`, and `faiss` imports lazy enough that OpenAPI/tests can import the FastAPI app without loading the full ML stack immediately.
- Fixed `BrainPackMemoryImporter` so it can call the current `MemoryService.add_memory(MemoryCreate(...))` signature as well as older keyword-based prototypes.
- Normalised brain-pack memory importance values from both `0.0-1.0` and `1-10` scales into Mozok's current `1-10` integer memory importance scale.
- Wired the main `BrainPackImportService` dry-run report to include a real memory import preview.
- Wired non-dry-run brain-pack memory import through `MemoryService`, so memories go through SQL, embeddings, and FAISS indexing.
- Fixed the standalone `scripts/import_brain_pack_memories.py` script to use the real `get_memory_service(db)` factory.

## Tests

- Added tests for the current `MemoryService.add_memory(MemoryCreate(...))` import path.
- Added tests for brain-pack memory importance normalisation.
- Added a dry-run scenario import test that checks `memory_import` preview reporting.
- Full suite result in the review environment: `120 passed, 3 skipped`.

## Notes

- The skipped tests are HTTP smoke tests and require a running local Mozok API.
- Pydantic class-based `Config` warnings remain non-blocking cleanup for a later patch.
- Brain-pack memory import runs through `MemoryService` after structural scenario import. This keeps embeddings and FAISS indexing correct, but those FAISS writes are not part of the structural scenario transaction wrapper.
